# 题目：多传感器数据融合与异常处理

（基础编程能力验证类）

**题目难度**：★★★☆☆

## 题目背景

在无人机姿态解算系统中，通常需要融合加速度计和陀螺仪的数据来获得稳定的角度估计。加速度计提供长期稳定的角度参考但易受高频噪声干扰，陀螺仪动态响应快但存在积分漂移。通过互补滤波或卡尔曼滤波可以融合两者优势，获得更准确的姿态角。

真实硬件环境中，传感器可能因电磁干扰、电源波动或物理冲击产生异常输出（如 NaN、超量程值），这些异常数据若直接参与融合计算，可能导致系统输出剧烈跳变甚至失控。本题目要求你在 STM32 嵌入式平台上实现一个鲁棒的传感器数据融合模块，能够实时处理加速度计和陀螺仪数据流，检测并处理异常输入，并在双模环境（QEMU 仿真 vs 硬件）下验证算法的一致性。

## 任务描述

模拟传感器数据通过两个全局数组提供（由测试平台每 10ms 更新一次）：

```c
/* 模拟传感器数据（由外部平台每10ms写入最新值） */
extern volatile float accel_data[3];   /* 加速度计: ax, ay, az (单位: g) */
extern volatile float gyro_data[3];    /* 陀螺仪: gx, gy, gz (单位: °/s) */

/* 你需要实现的融合输出 */
extern volatile float pitch;            /* 俯仰角输出 (度) */
extern volatile float roll;               /* 横滚角输出 (度) */
```

你需要实现以下核心函数：

```c
/* 初始化融合算法（重置滤波器状态） */
void fusion_init(void);

/* 每10ms被定时器中断调用一次，处理最新的传感器数据 */
void fusion_update(void);

/* 异常检测辅助函数（可选，但建议模块化） */
static int is_data_valid(float *accel, float *gyro);
```

### 具体要求

1. **数据融合算法（主路径，用于融合输出 pitch/roll）**  
   实现一阶互补滤波，公式如下：

   `angle = 0.98f * (angle + gyro * dt) + 0.02f * accel_angle`

   其中 `dt` 精确为 `0.01f`（10ms）。`accel_angle` 由加速度计计算：

   - `pitch_acc = atan2f(-ax, sqrtf(ay*ay + az*az)) * (180/π)`（度）
   - `roll_acc = atan2f(ay, az) * (180/π)`（度）

   俯仰角速率使用 `gx`，横滚角速率使用 `gy`（单位 °/s，与 `dt` 相乘后为度）。

2. **第二条估计路径（仅用于评测指标 RMSE_hw，不写入 pitch/roll）**  
   在**同一时刻、同一组传感器数据**下，用**纯加速度计角度**作为估计：

   - `pitch_hw = pitch_acc`
   - `roll_hw = roll_acc`

   与上式相同的 `pitch_acc`、`roll_acc` 定义。该路径不参与互补滤波状态更新。

3. **异常输入处理**

   - 检测 NaN、Inf：若任一传感器值为 NaN 或无穷大，丢弃本次更新，保持上一时刻角度输出。
   - 检测超量程：加速度计正常范围 ±2g，陀螺仪 ±250°/s，超出此范围的数据视为无效，同样丢弃本次更新。
   - 连续异常计数：若连续 5 次以上更新都因异常被丢弃，则强制调用 `fusion_init()` 复位滤波器状态（内部互补滤波状态及输出角按 `fusion_init` 定义清零）。

4. **双模一致性（课程叙述）**  
   代码需具备在 QEMU 仿真与 STM32 硬件上运行的结构；**本机评测**不连接实机，通过给定参考角与 RMSE 指标完成等价验证（见下文「评测补充说明」）。

5. **代码质量要求**

   - 禁止使用浮点库之外的动态内存分配（如 `malloc`）。
   - 必须处理三角函数相关异常输入（如避免对非法值使用 `acos`；本题主要使用 `atan2f`）。
   - 遵循 MISRA C 基本习惯：显式类型转换、避免可疑隐式转换、浮点字面量带 `f` 后缀等。

6. **并发安全**

   - `fusion_update` 在定时器中断中执行时，主循环可能读取 `pitch`/`roll`，因此这两个输出变量必须用 `volatile` 修饰。
   - 评测环境下由主循环顺序调用 `fusion_update` 模拟 10ms 节拍，仍须保留 `volatile` 以满足题意。

## 输入格式（课程叙述）

测试平台通过模拟数据提供加速度计与陀螺仪时间序列。真实系统可为两个文本日志；**本仓库评测**使用单文件格式（见「评测补充说明」）。

## 输出格式（课程叙述）

程序在**处理完全部采样**后，通过串口打印统计信息（精确格式见「评测补充说明」）。

## 样例（说明性）

**样例 1（正常融合）**  
加速度计与陀螺仪片段与参考文档一致时，输出形态示例：

```text
[FUSION] RMSE_sim: 0.087 deg
[FUSION] RMSE_hw: 0.092 deg
[FUSION] Diff: 5.7% (within spec)
[FUSION] Abnormal drops: 0 times (0.0% of total)
```

**样例 2（含异常输入）**  
加速度计某行出现 `NaN` 时，该时刻融合更新被跳过，输出保持前一时刻角度，最终报告含丢弃次数。

## 提示

- 互补滤波中的 `dt` 应精确为 `0.01f`（10ms）。
- 异常检测时使用 `isnan`、`isinf`（`<math.h>`）。
- 硬件上浮点运算可能较慢，应控制 `fusion_update` 耗时。
- QEMU 仿真可用脚本叠加噪声；评测数据由 `data/*.in` 提供。

## 评测补充说明（机判必读）

本节为在本平台 **裸机 UART 评测** 下的**强制性约定**，与 `data/*.in`、`data/*.out` 一致。

### 提交方式

- 学生**只提交一个 C 源文件**。
- 可使用 `baremetal/` 提供的 `uart_oj_rx_poll.h`，在 `main` 的循环中调用 `oj_uart_poll_step()`，并在 `UART_IRQHandler` 中按字节组帧为**一行文本**（以 `\n` 结束），再解析；亦可采用与标程等价的读行方式。
- 程序从 **UART 读入**（评测机将 `data/*.in` 全文经 UART 注入；标程通过 `oj_uart_poll_step()` + `UART_IRQHandler` 组行，**不要**再使用 `scanf`/stdin 与 IRQ 同时抢同一字节流）。
- 处理完毕后向 **UART 写出**（使用平台 `uart_write_byte` 或等价方式），共 **4 行**，与标准答案**全文一致**（判题程序对换行与行尾空格做规范化后做字符串比较）。
- **禁止**使用 `printf` 的 **`%f` / `%e` 等浮点格式**：在 8KB SRAM 的 `stm32vldiscovery` 裸机镜像上易撑爆栈与 RAM。应手写整数部分 + 小数位输出（与下文舍入规则一致），标程亦如此。

### 评测输入格式（单文件，对应 `data/*.in`）

1. 第一行：整数 `N`（采样点数，每点对应 10ms）。
2. 接下来 `N` 行：每行 **8 个浮点数**，空格分隔：

   `ax ay az gx gy gz ref_pitch ref_roll`

   - 前 6 个为传感器；`ref_pitch`、`ref_roll` 为该时刻**参考俯仰/横滚（度）**，仅用于 RMSE 计算，**不参与**滤波器更新。
3. 允许出现 `nan`、`inf` / `infinity` 等文本（大小写不敏感）。**不推荐**在裸机上使用 `strtof`：会拖入 newlib 大块 RAM 与深栈；可自行实现词法解析，或与标程一样解析十进制小数、`nan`、`inf`/`infinity`。

### 评测输出格式（必须完全一致）

共 4 行（每行以 `\n` 结束）：

1. `[FUSION] RMSE_sim: ` + **三位小数** + ` deg`。RMSE 为非负；打印值取 `x = RMSE_sim`，令 `k = (uint32_t)(x * 1000.0f + 0.5f)`，输出整数部分 `k/1000`、小数点、再输出三位十进制小数 `k % 1000`（不足三位左侧补 `0`）。
2. `[FUSION] RMSE_hw: ` + **三位小数** + ` deg`（同上，对 `RMSE_hw`）。
3. `[FUSION] Diff: ` + **一位小数** + `% (` + `within spec` 或 `not within spec` + `)`  
   - 先计算  
     `diff_pct = |RMSE_sim - RMSE_hw| / max(RMSE_sim, RMSE_hw, 1e-6f) * 100.0f`  
   - 若 `diff_pct <= 10.0f` 则打印 `within spec`，否则 `not within spec`。  
   - `Diff` 行的百分数：对 `diff_pct` 取 `t = (uint32_t)(diff_pct * 10.0f + 0.5f)`，输出 `t/10`、`.`、`t%10`；若为负数先输出 `-` 再对绝对值同上（本题数据下一般为非负）。
4. `[FUSION] Abnormal drops: ` + **十进制整数** + ` times (` + **一位小数** + `% of total)`  
   - `Abnormal drops` 为因异常被丢弃的**更新次数**（与题意一致）。  
   - 百分比分母固定为输入第一行的 **`N`**：`100.0f * (float)drop_count / (float)N`。若 `N == 0`，则打印 `0 times (0.0% of total)`。

### RMSE 定义（机判版）

- 设每一步在**未丢弃**且已执行互补滤波更新后，得到融合角 `pitch`、`roll`；同时由**当前** `ax,ay,az` 按题意计算 `pitch_acc`、`roll_acc`（与 RMSE_hw 路径一致）。
- **有效步**：该次 `fusion_update` **未被丢弃**的采样步。
- 对每一个有效步，定义：

  `e_sim = (pitch - ref_pitch)^2 + (roll - ref_roll)^2`  
  `e_hw = (pitch_acc - ref_pitch)^2 + (roll_acc - ref_roll)^2`

- 若有至少一个有效步：

  `RMSE_sim = sqrt( (所有有效步 e_sim 之和) / (有效步数) )`  
  `RMSE_hw = sqrt( (所有有效步 e_hw 之和) / (有效步数) )`

- 若**没有任何有效步**：`RMSE_sim = 0`，`RMSE_hw = 0`，`Diff` 行为 `0.0% (within spec)`，`Abnormal drops` 仍按上式统计（分母为 `N`，`N==0` 时见上文）。

### 浮点判题

- 判题为**规范化后的全文字符串相等**，须与标程使用**相同的舍入与字面量**（上文三位/一位小数规则），避免多余空格或额外行。
- `data/*.out` 由 `tools/gen_expected.py` 生成，其输入解析与标程 **C 代码逐 token 一致**（同一套十进制 → `float32` 步骤）；若自行造数据，勿用「双精度 `float()` 再转 float32」代替标程解析，否则 RMSE 可能差 **0.001** 而 WA。
