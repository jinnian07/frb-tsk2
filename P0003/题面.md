# 题目：静态内存池管理器（内存安全类）

**题目难度：** ★★☆☆☆  

## 题目背景

在无人机飞控或卫星载荷计算机中，通常禁止 `malloc`/`free`，以避免堆碎片与分配失败；同时需防范越界、重复释放与野指针。本题要求在裸机环境下实现**固定块静态内存池**，配合金丝雀与魔数做安全检测，并通过 UART 命令验证行为。

## 任务描述

### 内存布局

- 共 **8** 个块，每块物理 **32** 字节，连续存放在静态数组 `memory_pool` 中。
- 单块布局（低地址 → 高地址）：**前 4 字节金丝雀** → **中间 24 字节用户区** → **后 4 字节金丝雀**。
- 金丝雀值为 **`0xDEADBEEF`**，按**小端**写入 4 字节。
- `mem_pool_alloc` 成功时返回指向**用户区首字节**的指针（不指向金丝雀）。

### 控制块

```c
#define BLOCK_SIZE      32
#define BLOCK_COUNT     8
#define POOL_SIZE       (BLOCK_SIZE * BLOCK_COUNT)

static uint8_t memory_pool[POOL_SIZE];

typedef struct {
    uint8_t used;      /* 0=空闲, 非0=已分配 */
    uint8_t *ptr;      /* 指向该块用户区首地址 */
    uint16_t magic;    /* 魔数 0xA5A5，用于完整性检查 */
} block_ctrl_t;

static block_ctrl_t block_ctl[BLOCK_COUNT];
```

- **禁止**使用 `malloc` / `free`，所有存储来自 `memory_pool` 与 `block_ctl`。
- 分配策略：**首次适应**，自低索引起找第一个空闲块。

### 必须实现的 API

```c
void mem_pool_init(void);
void *mem_pool_alloc(void);
int mem_pool_free(void *ptr);
int mem_pool_check(void *ptr, size_t write_size);
```

语义说明：

| 函数 | 行为 |
|------|------|
| `mem_pool_init` | 初始化控制块与池；每块写入金丝雀；用户区清零；魔数仅在分配时写入。 |
| `mem_pool_alloc` | 分配一块用户区；成功返回指针，失败返回 `NULL`。 |
| `mem_pool_free` | 释放；成功返回 `0`，失败返回 `-1`。失败原因包括：指针不在池内合法用户区、重复释放、魔数不匹配、金丝雀被破坏。 |
| `mem_pool_check` | 若 `ptr` 为当前已分配块的用户区首地址、魔数正确、金丝雀完好，且 **`write_size <= 24`**，返回 `1`，否则返回 `0`。 |

### 临界区

`mem_pool_alloc`、`mem_pool_free` 在修改 `block_ctl` 及块元数据期间须关中断（如 `cpsid i` / `cpsie i`），临界区尽量短。

## UART 交互协议（评测）

- 输入为 **UTF-8 文本**，**一行一条命令**，行尾为 `\n`。
- 输出仅通过 **`uart_write_byte`** 打印下列**固定字符串**（含换行），须与标程**逐字一致**（全角/半角、标点均一致）。

### 命令

| 命令 | 说明 |
|------|------|
| `ALLOC` | 分配一块；按分配先后顺序得到编号 `0`～`7`。 |
| `WRITE <id> <hex> ...` | 空格分隔的两位十六进制字节序列，写入编号 `id` 对应指针起始处；**无论长度多少，只要命令解析完成即打印 `WRITE OK`**（可能破坏金丝雀）。 |
| `FREE <id>` | 释放编号 `id`。对**同一编号**在已成功释放后再次 `FREE`，须输出 `FREE ERROR: 重复释放`。 |
| `CHECK <id>` | 对 `write_size == 0` 调用 `mem_pool_check`。 |
| `RESET` | `mem_pool_init()`，并清空交互层「编号→指针」映射。 |

### 输出（固定文案）

- `ALLOC OK: <id>` 或 `ALLOC FAIL`
- `WRITE OK`
- `WRITE ERROR: 非法指针`（编号无效或未分配）
- `CHECK OK` 或 `CHECK ERROR: 内存损坏`
- `FREE OK`  
- `FREE ERROR: 非法指针`  
- `FREE ERROR: 重复释放`  
- `FREE ERROR: 内存越界，金丝雀值被破坏`  
- `RESET OK`

**说明：** 越界破坏由 **`FREE`**（及 **`CHECK`**）通过金丝雀检测；**`WRITE` 不因长度超限而报 `WRITE ERROR`**（除非 `id` 非法）。

## 样例

**样例 1（正常）**

输入：

```
ALLOC
WRITE 0 11 22 33 44
CHECK 0
FREE 0
```

输出：

```
ALLOC OK: 0
WRITE OK
CHECK OK
FREE OK
```

**样例 2（重复释放）**

```
ALLOC
FREE 0
FREE 0
```

输出：

```
ALLOC OK: 0
FREE OK
FREE ERROR: 重复释放
```

**样例 3（越界写坏金丝雀）**

写入超过 24 字节用户数据将覆盖尾部金丝雀；`WRITE` 仍输出 `WRITE OK`，`FREE` 报错。

## 实现提示

- 平台与 **P0001/P0002** 相同：在 `main` 中循环调用 `oj_uart_poll_step()`，在 `UART_IRQHandler` 中调用 `UART_ReceiveByte()` 按行组帧。
- 编号 `id` 与池内块下标无关，由**分配顺序**决定；`RESET` 后重新从 `0` 编号。
- 使用 `volatile` 协调行缓冲与主循环，避免与中断并发问题。

## 评测方式

- 裸机模式：`cortexm_baremetal_uart`，输入来自 `data/*.in`，比对 UART 输出与 `data/*.out`（规范化后全文相等）。
- **请勿**将 `.in` 写成整文件仅含两位十六进制 token，否则会被判题机当作二进制字节流而非文本命令。
