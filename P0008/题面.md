# 题目八：低功耗数据采集与功耗分析

**难度**：★★★☆☆  
**评测模式**：裸机 QEMU + UART（`cortexm_baremetal_uart`），与 P0001 / P0002 相同链路。

## 背景

嵌入式产品在电池供电场景下需要在「采集传感器数据」与「功耗」之间折中。本题模拟：先从 UART 读入硬件电流表测得的两档电流（正常模式 / 低功耗模式），再在**正常模式**与**低功耗模式**下各输出 **3 次**温度采样行，最后按固定格式打印功耗对比报告（含仿真参考值对比）。

## 输入（UART 文本）

评测机将 `data/*.in` **全文**经 UART 注入（UTF-8 文本，行尾可为 `\n` 或 `\r\n`）。

- **第 1 行**：正常模式平均电流，单位 mA，**一位小数**（如 `25.3`）。
- **第 2 行**：低功耗模式平均电流，单位 mA，**一位小数**（如 `1.2`）。

## 输出（UART 文本，须完全一致）

1. **正常模式阶段**：依次输出 3 行（顺序为 `k=0,1,2`）  
   `Temp: 25.1 C` / `Temp: 25.2 C` / `Temp: 25.3 C`（即小数位循环 1、2、3）。

2. **低功耗模式阶段**：同样再输出上述 3 行（格式相同）。

3. **固定报告块**（标题、字段名、数值格式、换行须与标程一致），包括：
   - 两行平均电流（mA，一位小数）；
   - **降幅** `Reduction:` 为百分比，**一位小数**，算法为：先以**十分之一毫安**整数计算 \((I_{norm}-I_{lp})/I_{norm}\)，再**四舍五入**到一位小数（与标程一致）；
   - **Simulation vs Hardware difference** 两段：仿真正常电流 **24.8 mA**、仿真低功耗 **1.1 mA**，与读入的硬件值比较，输出 **带符号** 的百分比差（一位小数 + `%`），四舍五入规则与标程一致；
   - 最后一行 **Analysis:** 固定为：  
     `QEMU not cycle-accurate; UART and LDO not in energy model; meter noise on HW.`

## 实现要点

- 使用 `baremetal/uart_oj_rx_poll.h`：在循环中调用 `oj_uart_poll_step()`，在 `UART_IRQHandler` 中通过 `UART_ReceiveByte()` **按字节组行**（以 `\n` 结束），**不要**与 `scanf`/stdin 混用同一字节流。
- 两次采样阶段之间可用软件延时等效「采样间隔」；在 QEMU 判题环境下**不要**依赖 `__WFI` 唤醒（无可靠中断时可能无法继续）。
- 禁止使用 `malloc`；注意缓冲区边界与 MISRA 习惯。

## 测试数据

题库含 **10** 组测试点：`train01`～`train10`。
