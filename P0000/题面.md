# UART 串口发送字符串

## 描述

## 评测模式

**裸机 QEMU + UART**（`cortexm_baremetal_uart`）：与 P0001、P0002 相同链路。评测机将 `data/*.in` 经 **UART 注入** 输入，采集 **USART1 发送** 的字节流，与 `data/*.out` 比对（规范化后全文相等）。

## 题目背景

串口是嵌入式系统与上位机通信的最基础手段之一。本题要求掌握 **USART 初始化** 与 **按字节发送** 的流程，在程序启动后向上位机发送一条固定问候字符串。

## 仿真与教学说明

- 教学材料中常出现 **STM32F4** 与 `stm32f4xx.h`；本 OJ 实际在 **QEMU `stm32vldiscovery`**（**Cortex-M3**）上运行，USART1 外设基址与波特率计算与课程裸机运行时一致（系统时钟 **24 MHz**，与 `baremetal/uart1_qemu.c` 一致）。
- 你必须按本题参数配置 **USART1**，并通过运行时提供的 **`uart_write_byte`** 或等效方式逐字节送出数据（**不要使用 `printf`**，以免占用过多 RAM / 引入 newlib 依赖）。

## 要求

## 任务描述

### 串口参数

| 项目     | 值        |
|----------|-----------|
| 外设     | USART1    |
| 波特率   | 115200    |
| 数据位   | 8         |
| 校验     | 无        |
| 停止位   | 1（8N1）  |

### 必须实现的 API

```c
void uart_init(void);
void uart_send_string(const char *s);
```

- **`uart_init`**：完成 USART1 的 **115200、8N1** 配置，并使能发送（接收可按平台需要一并使能；本运行时模型下评测机会向 UART 注入输入，使能 **UE + TE + RE** 与参考实现一致即可）。
- **`uart_send_string`**：从 `s` 指向的字符串首字节开始，依次发送每一个字符，直到遇到 **`'\0'`** 停止；若 `s` 为 **空指针**，函数直接返回、不发送任何字节。

### `main` 中的逻辑

在 **`main`** 中：

1. 调用 **`uart_init()`**；
2. **在发送问候之前**，须通过 **`oj_uart_poll_step()`** 与 **`UART_IRQHandler`** 的配合，**等到至少收到 UART 上的 1 个来自评测机的字节**（评测机会在连接后注入 `data/*.in` 对应的数据，空输入也会补发换行，故一定会到达至少一字节）。  
   **原因（必读）**：裸机评测使用 QEMU 的 **TCP `server,nowait` 串口后端**；若程序一上电就立刻向 USART1 发送，此时上位机往往尚未 `connect`，**先发出去的 TX 字节会丢失**，导致评测采集为空、全部 **WA**。
3. 收到上述至少一字节后，调用 **`uart_send_string()`**，发送字面字符串（半角、逐字一致）：**`Hello, Relilearn!`**
4. 再发送一个 **换行符** **`'\n'`**，便于串口终端换行显示。

之后进入主循环，并**持续调用** **`oj_uart_poll_step()`**（需 `#include "uart_oj_rx_poll.h"`），以便继续排空评测机注入的 UART 数据（与 P0001 等题一致）。

实现提示：可定义 **`void UART_IRQHandler(void)`**（强符号，覆盖 `uart_oj_rx_poll.c` 中的 weak 空实现），在每次被调用时置位一个 **`volatile` 标志**；在 `main` 里先 **`while (!标志) { oj_uart_poll_step(); }`**，再执行第 3、4 步。

## 平台符号与 `uart_init` 说明

链接时，`baremetal/uart1_qemu.c` 中提供了一个 **弱符号（weak）** 默认的 `uart_init`，供未自定义该函数的题目使用。**本题要求你提交源码中提供强符号 `void uart_init(void)`**，覆盖弱默认实现，完成上述配置；**不得**与运行时产生重复定义的链接错误。

启动代码会在进入 **`main` 之前**调用一次 `uart_init()`；你在 **`main` 内再调用一次**属于重复初始化，只要寄存器配置一致即为幂等，符合本题要求。

## 输入格式

评测机将 **`data/*.in` 全文** 按 UTF-8 经 UART 注入（行尾可为 `\n` 或 `\r\n`）；若文件为空，仍会注入至少一个换行以满足管道行为。**本题不要求**根据输入内容改变输出：无论输入为何，程序均应在**收到至少一字节注入数据之后**再发送上述固定字符串与换行。

## 输出格式

通过 UART 发送的字符序列，规范化后与标准答案 **全文一致**（判题程序会统一换行、去除行末空格与文末空行，参见 `judger/core/oj_engine.py` 的 `normalize_text`）。

**标准输出内容**（含一行结束）为：

```text
Hello, Relilearn!
```

即：字符串 **`Hello, Relilearn!`** 后紧跟一个 **`\n`**。

## 提示

- **切勿**在 `main` 开头、未与评测机完成「至少一字节 RX」同步前就发送问候行，否则在 QEMU 评测环境下极易 **WA**（原因见上文「`main` 中的逻辑」）。
- 发送每个字节前需等待发送数据寄存器空（**TXE**）再写入 **DR**，与 `uart_write_byte` 实现思路一致；也可直接调用 **`extern void uart_write_byte(uint8_t ch);`** 发送单字节。
- **BRR** 计算：`brr = (SYSCLK_HZ + baud * 8) / (baud * 16)`（整数四舍五入），其中 **SYSCLK_HZ = 24_000_000**。
- **CR1**：置位 **UE、TE**（及本题参考中的 **RE**），数据位 8、无校验、1 停止位为 QEMU 模型默认/常用配置。
- 主循环务必包含 **`oj_uart_poll_step()`**，否则注入字节可能堆积，且与题库统一评测模型一致。
